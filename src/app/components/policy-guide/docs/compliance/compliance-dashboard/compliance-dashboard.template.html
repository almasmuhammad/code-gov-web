Agency Compliance

The Source Code Policy requires agencies to perform specific one-time and recurring tasks in order to be considered compliant. Currently, agencies are evaluated on whether they've completed three tasks:

  1. Revised agency policy: agencies must update their policies to be consistent with the Federal Source Code Policy.
  2. Completed code inventory: agencies must publish a complete inventory of their source code in the prescribed format on their website.
  3. Met open source objective: agencies must open source 20% or more of their new custom code created after August 2016.

Agencies are rated as fully, partially, or non-compliant with these factors - green, yellow or red. An agency's overall compliance is green only if both of the factors are green.
